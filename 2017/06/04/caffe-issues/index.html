安装或运行Caffe遇到的一些问题

发表于 2017-06-04 | 阅读次数

所有问题

形如“undefined reference to TIFFRGBAImageOK@LIBTIFF_4.0”的问题。

  1. 参考opencv论坛的方法
  2. 如果装了anaconda的话，可能与anaconda的tiff库冲突，使用下列命令

No module named caffe

ImportError: libcudart.so.*.0: cannot open shared object file: No such file or directory

no module named cv2

  1. 没装python-opencv

  2. 或者

  3. 或者将cv2.so copy到anaconda目录下

version `GLIBCXX_3.4.21’ not found

File already exists in database: caffe.proto

  1. 卸载protobuf，从’https://github.com/google/protobuf'重新下载安装protobuf

  2. 重装opencv

registry.count(type) == 0 (1 vs. 0) Layer type Convolution already registered.

  1. 有可能是在同时使用不同版本的caffe，注释掉后调用的caffe(include/layer_factory.hpp),或者全部caffe里面都注释，重新make。

fatal error: caffe/proto/caffe.pb.h: No such file or directory

/usr/lib/x86_64-linux-gnu//libunwind.so.8: undefined reference to ‘lzma_index_uncompressed_size@XZ_5.0’

  1. 查看liblzma的位置

  2. 从其他没问题的电脑copy所有liblzma*到上面得到的位置（网上其他方法试了不行，这个可以）。然后执行

build_release/lib/libcaffe.so: undefined reference to ‘google::protobuf::io::CodedOutputStream::WriteVarint64ToArray(unsigned long long, unsigned char*)’

  1. 用gcc5或者更新的编译器重新编译

cv::imread(cv::String const&, int)’ .build_release/lib/libcaffe.so: undefined reference to cv::imencode(cv::String const&, cv::_InputArray const&, std::vector&, std::vector const&)’

  1. 存在多个版本的opencv,查看使用哪个位置的opencv
